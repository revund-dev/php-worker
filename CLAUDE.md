# revund/workers/php

PHP AST sidecar for Revund. Implements the universal `revund.worker.v1.Worker` contract
(same one ts-worker and ruby-worker speak); parses `.php` files via nikic/PHP-Parser and
returns the universal AST view the structural detectors consume.

## Read first

See `/CLAUDE.md` for product context.
See `/proto/worker/v1/worker.proto` for the wire contract every worker speaks.

## Stack

- **Runtime:** PHP 8.1+
- **AST library:** `nikic/php-parser` — the de facto PHP AST library; PHPStan,
  Psalm, and every serious PHP static analyzer build on it.
- **gRPC:** `grpc/grpc` (PECL extension + Composer package) + `google/protobuf`.

The proto contract lives at the workspace root (`/proto/worker/v1/worker.proto`).
`composer proto` runs protoc with `--proto_path=../../proto`.

## RPCs

The universal `revund.worker.v1.Worker` service:

| RPC              | Status      | Purpose |
|------------------|-------------|---------|
| `Describe`       | required    | Self-identifies: name="php-worker", languages=["php"], capabilities=["parse"]. |
| `Health`         | required    | Liveness probe. |
| `Parse`          | required    | Returns the universal `ParsedFile` (imports + decls + functions + concerns). |
| `ResolveSymbols` | unimplemented | Bot's caller checks `capabilities` and skips this call. |
| `RunDiagnostics` | unimplemented | Same. PHPStan/Psalm integration is a future capability. |

## Current status

**Full nikic/PHP-Parser implementation shipped.** `Parser.php` walks the AST and
emits the universal `ParsedFile` shape — imports, top-level decls, functions
(with hash + canonical_hash + blocks), and concern evidence. Cross-language
canonical hashes line up with ts-worker and ruby-worker so the structural
detectors can cluster shared shapes across languages.

The traversal:

1. Parses via `ParserFactory::createForHostVersion()`.
2. Resolves names with the `NameResolver` visitor so `use` aliases land on
   their fully-qualified targets.
3. Collects imports (`Use_` / `UseUse`), decls (`Class_` / `Interface_` /
   `Trait_` / `Enum_` / `Function_`), functions+methods with cyclomatic
   complexity and hash variants, and concern evidence (Session::/Cache::/
   Config:: → state, Http::/Guzzle/curl → network, DB::/Eloquent →
   dataaccess, etc.).
4. On `PhpParser\Error`, captures into `parse_error` and returns whatever
   partial data the parser salvaged.

## Build + run

```bash
# Install deps
composer install

# Generate proto stubs (requires protoc + grpc_php_plugin)
composer run proto

# Start the worker
composer run start
# OR explicitly:
REVUND_PHP_WORKER_PORT=50052 bin/php-worker
```

The server emits `ready: 0.0.0.0:<port>` once `addHttp2Port` succeeds. Spawning
parents grep for that line to synchronize.

## Ports

Convention: each first-party sidecar gets a port offset from 50051.

- ts-worker → 50051
- php-worker → 50052
- ruby-worker → 50053

Override via `REVUND_PHP_WORKER_PORT`.

## Error handling

- Sidecar must never crash the gRPC server on a single bad request — every
  handler wraps its work in try/catch and surfaces errors via `parse_error`
  (per-file) or RPC `INTERNAL` status (whole-batch fatal).
- If `nikic/php-parser` itself isn't installed, log to stderr and exit
  non-zero. The bot falls back to NopClient on health-check failure.

## Testing

